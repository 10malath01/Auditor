# 🔐 برنامج تدقيق تقارير اختبار الاختراق الآلي
## AI-Based Automated Auditing of Penetration Testing Reports

---

## 📌 نظرة عامة على المشروع

برنامج متقدم يقوم بـ **تدقيق آلي شامل** لتقارير اختبار الاختراق (Penetration Testing Reports) بهدف التأكد من **اكتمال التوثيق والالتزام بمعايير إدارة الثغرات**.

البرنامج لا يختبر الأنظمة، بل يختبر **جودة التوثيق والالتزام بالمتطلبات**، مما يوفر وقتاً كبيراً على فريق التدقيق اليدوي.

---

## 🎯 الأهداف الرئيسية

### 1️⃣ التدقيق الآلي للعناصر السبعة (7 Audit Elements)

البرنامج يتحقق من وجود **7 عناصر أساسية** في كل ثغرة:

| # | العنصر | الوصف | الأهمية |
|---|--------|-------|--------|
| **E1** | **التعريف** (Definition) | تعريف واضح للثغرة وماهيتها | ⭐⭐⭐⭐⭐ |
| **E2** | **الوصف** (Description) | شرح تفصيلي للثغرة وتأثيرها | ⭐⭐⭐⭐⭐ |
| **E3** | **الموقع** (Location) | أين توجد الثغرة (URL, IP, Port) | ⭐⭐⭐⭐ |
| **E4** | **الأجهزة المتضررة** (Affected Assets) | ما هي الأنظمة المتأثرة | ⭐⭐⭐⭐ |
| **E5** | **تصنيف الخطورة** (Severity) | مستوى خطورة الثغرة (Critical/High/Medium/Low) | ⭐⭐⭐⭐⭐ |
| **E6** | **حالة المعالجة** (Remediation Status) | هل تم إصلاح الثغرة؟ | ⭐⭐⭐⭐ |
| **E7** | **طريقة المعالجة** (Remediation Method) | كيف تم إصلاح الثغرة؟ | ⭐⭐⭐⭐ |

### 2️⃣ تحليل عنصر المعالجة بـ NLP

البرنامج يستخدم **معالجة اللغات الطبيعية (NLP)** لـ:

- **استخراج نوع الهجوم** (Attack Type) - مثل SQL Injection, XSS, etc.
- **تحديد الحلول المقترحة** (Defense Recommendations) - الحلول المعروفة لكل نوع هجوم
- **مقارنة الحل المطبق** - هل الحل المستخدم في التقرير يطابق أفضل الممارسات؟

### 3️⃣ نموذج تعلم آلي مدرب

يتم تدريب نموذج **Random Forest** على:

- **بيانات من Kaggle** - مجموعة بيانات التهديدات الأمنية العالمية (2015-2024)
- **تصنيف الخطورة** - التنبؤ بمستوى خطورة الثغرة
- **استخراج الحلول** - التعرف على أنواع الهجمات والدفاعات المناسبة

### 4️⃣ نظام الأوزان (Weighting System)

يتم حساب **نسبة الالتزام المرجحة** بناءً على أهمية كل عنصر:

```
النسبة المرجحة = (E1×20% + E2×20% + E5×25% + E6×15% + E3×10% + E4×5% + E7×5%)
```

حيث:
- **E5 (Severity)** و **E1, E2 (Definition, Description)** الأكثر أهمية
- **E7 (Remediation Method)** الأقل أهمية نسبياً

## 🛠️ المكونات الرئيسية

### 1. **PDF Extractor** (`pdf_extractor.py`)
استخراج النصوص من ملفات PDF بشكل دقيق

### 2. **Vulnerability Segmenter** (`vulnerability_segmenter.py`)
تقسيم التقرير إلى ثغرات منفصلة باستخدام أنماط البحث

### 3. **Audit Engine** (`audit_engine.py`)
التدقيق الأساسي للعناصر السبعة باستخدام Regex والكلمات المفتاحية

### 4. **NLP Remediation Analyzer** (`nlp_remediation_analyzer.py`) - *جديد*
تحليل عنصر المعالجة واستخراج نوع الهجوم والحلول

### 5. **ML Model Trainer** (`ml_model_trainer.py`) - *محسّن*
تدريب نموذج Random Forest على بيانات Kaggle

### 6. **Weighted Compliance Calculator** (`weighted_compliance.py`) - *جديد*
حساب نسبة الالتزام المرجحة بناءً على أهمية العناصر

### 7. **Report Generator** (`report_generator.py`)
إنشاء تقارير مفصلة بصيغ مختلفة

### 8. **Streamlit App** (`app.py`) - *جديد*
واجهة تفاعلية سهلة الاستخدام

الثغرة 1: 85% (SQL Injection)

الثغرة 4: 88% (Missing Headers)
الثغرة 5: 75% (Outdated Dependencies)

## 🔬 تقنيات المعالجة المستخدمة

### 1. **Regular Expressions (Regex)**
- البحث عن الأنماط في النصوص
- استخراج المعلومات المهيكلة

### 2. **Natural Language Processing (NLP)**
- فهم المعنى من النصوص
- استخراج الكيانات (Entities)
- تحليل المشاعر والسياق

### 3. **Machine Learning (ML)**
- **TF-IDF**: تحويل النصوص إلى أرقام
- **Random Forest**: تصنيف وتنبؤ
- **Feature Engineering**: استخراج الميزات المهمة

### 4. **Text Preprocessing**
- تنظيف النصوص
- إزالة الكلمات الشائعة
- تطبيع الأحرف
